# LangChain框架的Agent入门

**文档类型**：lesson  

## 教学目标

1. 知识与技能：学生能够理解LangChain框架中Agent的基本概念与作用；掌握LangChain Agent的最简单代码结构，包括初始化、调用和返回格式；能够独立完成LangChain的环境安装与配置，并运行一个基础Agent示例。
2. 过程与方法：通过从零开始的代码构建与演示，培养学生分析LangChain官方文档、动手配置和调试代码的能力。
3. 情感态度与价值观：激发学生对人工智能应用开发工具LangChain的兴趣，体验使用框架简化开发流程的效率，树立在新技术学习中实践验证的观点。

*时长：2 分钟*

## 导入

【提问与展示引入】 教师播放一个小视频或展示一个简单Demo：一个能够根据用户自然语言指令（比如“查询今天北京的天气”）自动调用网络搜索工具的AI助手。然后提问："这个看起来智能的助手背后可能需要哪些技术支撑？如何快速构建这样的AI应用？" 引导学生思考传统代码与AI驱动的自动任务执行的区别。由此引出能够自主规划、使用工具来完成任务的高层次组件——Agent，并介绍LangChain框架在构建此类应用中的优势（集成、模块化）。明确本节课任务：从零开始，配置环境并写出第一个最简单的LangChain Agent。

*时长：5 分钟*

## 新授

【第一环节：环境配置（约10分钟）】
1. 前提条件说明：需要Python环境（建议3.8+）、pip包管理工具，以及一个可用的OpenAI API密钥（或其他LLM提供商的密钥）。
2. 安装命令演示与讲解：
   - 创建并激活Python虚拟环境（可选但推荐）。
   - 核心安装：`pip install langchain langchain-openai`。
   - 可选工具安装示例：`pip install google-search-results` 用于后续可能用到的搜索工具，本节课仅作介绍。
3. 环境验证：在Python交互环境中，尝试`import langchain`，若无报错则初步成功。
4. 配置API密钥：演示如何将密钥设置为环境变量（如`os.environ["OPENAI_API_KEY"] = "sk-..."`），强调安全注意事项。

【第二环节：Agent核心结构讲解与编码（约20分钟）】
1. 概念讲解：
   - Agent是什么？可以理解为“大脑”（大语言模型） + “工具” + 决定使用哪个工具的“决策逻辑”。
   - 最简单的Agent流程：用户输入 -> Agent（分析输入，内部调用模型思考）-> 决定使用工具/直接回答 -> 返回结果。
2. 代码分步构建（使用最简单的“零-shot反应”Agent和单个预定义工具）：
   a. 导入必要模块：`from langchain.agents import initialize_agent, AgentType`， `from langchain.tools import Tool`， `from langchain_openai import OpenAI`。
   b. 初始化大语言模型（LLM）：`llm = OpenAI(temperature=0, openai_api_key=你的密钥)`。说明temperature参数作用。
   c. 定义一个非常简单的工具函数及Tool对象（示例：一个计算字符串长度的函数）：
      ```python
      def get_string_length(input_str: str) -> str:
          return f"The length of the string is {len(input_str)} characters."
      tools = [
          Tool(
              name="String Length Calculator",
              func=get_string_length,
              description="Useful for when you need to calculate the length of a string. Input should be a string."
          )
      ]
      ```
      强调Tool的三个关键属性：name, func, description。
   d. 初始化Agent：`agent = initialize_agent(tools, llm, agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION, verbose=True)`。
      - 解释参数：tools列表，llm，agent类型（ZERO_SHOT_REACT_DESCRIPTION是最基础的一种），verbose=True便于观察内部思考过程。
   e. 运行Agent：`result = agent.run("What is the length of the word 'hello'?")` 或 `agent.invoke("计算'hello'的长度")`。
   f. 展示输出结果，带领学生观察verbose模式下Agent的“Thought”（思考）、“Action”（选择工具）、“Observation”（工具返回）、“Final Answer”的完整链式步骤。

*时长：30 分钟*

## 巩固练习

【动手实践与调试】
1. 任务一：让学生将老师演示的代码在自己的环境中完整运行一遍，确保得到正确结果。
2. 任务二（扩展）：修改工具函数，例如，创建一个将输入字符串转换为大写的工具，并更新Tool的描述。然后让Agent运行一个新问题，如“把 'langchain' 变成大写字母”。观察Agent如何根据问题选择正确的工具。
3. 故障排查指导：提供常见错误列表及解决方法参考（如：ModuleNotFoundError检查安装；API密钥错误；网络超时等）。教师巡视，进行个别指导。

*时长：6 分钟*

## 小结作业

【课堂小结】
1. 回顾关键步骤：环境安装 -> 导入模块 -> 定义工具(Tool) -> 初始化LLM -> 组装Agent -> 运行。
2. 强调核心概念：Agent是利用LLM协调工具执行任务的智能体，本节课演示了最简单、最基础的构建流程。
3. 提出思考：本节课的Agent只有一个简单工具，如果面对复杂任务需要多个工具（如计算器、搜索、数据库查询），Agent如何知道选择哪个？这依赖于模型的“思考”和工具的清晰描述（description）。

【布置作业】
1. 基础作业：记录环境配置步骤和完整代码，并尝试向Agent提问一个需要结合两个不同工具（如先计算长度再转换为大写）的问题，观察并记录其执行过程与结果（若无法自动连续使用，则分别测试两个工具的正确性）。
2. 探索作业（选做）：阅读LangChain官方文档中关于Agent和Tool的更多类型，尝试为Agent添加一个真正的网络搜索工具（需额外API），或了解其他Agent类型（如STRUCTURED_CHAT）。

*时长：2 分钟*
