# 面向对象的程序设计第一课

**文档类型**：lesson  
**学科**：计算机java课程  
**年级**：大二  

## 教学目标

1. 知识与技能：理解面向对象编程中“封装”特性的基本概念、特点及意义。掌握在Java中实现封装的基本语法（private、public、getter、setter）。
2. 过程与方法：通过分析一个简单需求（如描述一个“学生”），对比面向过程与面向对象的解决思路，体会封装带来的数据保护和管理便利性。
3. 情感态度与价值观：初步建立用面向对象思想解决问题、编写健壮和安全代码的意识。

*时长：5 分钟*

## 导入

1. 教师提问引导（3分钟）：大家之前接触过哪种编程范式？是否有过这样的经历：程序里一个变量的值，在某个地方被意外地修改了，导致整个程序出现难以排查的错误？例如，我们有一个表示年龄的变量`age`，程序其他地方如果直接写`age = -100;`，程序会出问题吗？
2. 教师展示一个用面向过程方式模拟学生信息的代码片段（如定义了一个全局变量`studentAge`用于存储年龄），并让同学观察潜在的风险。
3. 引出话题：有没有一种方法，能约束外部代码如何访问这些数据，既能保护数据，又能让数据的使用更规范、更安全？——这就是今天要学习的核心概念：封装。

*时长：5 分钟*

## 新授

（第一部分：封装的概念与特点，5分钟）
1. **定义**：封装（Encapsulation）是面向对象的三大特性之一。它指的是将对象的属性（数据）和行为（方法）包装在一起，形成一个独立的单元；同时，对外隐藏对象的内部实现细节，仅通过公共的接口与外界交互。
2. **比喻**：就像一个“遥控器”。我们不关心遥控器内部的复杂电路（属性），只需要知道按哪些按钮（公共方法）可以实现换台、调音量等功能。
3. **特点**：
    - **信息隐藏**：核心特点。将不想被外界直接访问的细节（通常指的是属性）隐藏起来。
    - **数据保护**：提供公开的方法来控制对内部数据的访问和修改，可以在方法中添加合法性检查，从而保证了数据的有效性和安全性。
    - **高内聚**：将与数据和操作数据的方法紧密关联的代码组织在一起，提高了代码的内聚性。

（第二部分：如何在Java中构建具有封装特性的代码，12分钟）
1. **使用`private`关键字隐藏属性**：通过将成员变量（属性）声明为`private`，使其仅能在定义它的类内部被访问。
   示例：`private String name;`
2. **使用`public`方法（访问器和修改器）提供访问接口**：
   - Getter方法（访问器）：以`get`开头，用于获取属性值。
   - Setter方法（修改器）：以`set`开头，用于设置属性值，并可在其中进行数据验证。
3. **示例代码演示（以Student类为例）**：
   ```java
   public class Student {
       // 1. 使用private封装属性
       private String name;
       private int age;
       private String studentId;

       // 2. 提供公共的getter方法（访问器）
       public String getName() {
           return this.name;
       }
       public int getAge() {
           return this.age;
       }

       // 3. 提供公共的setter方法（修改器），可在其中加入控制逻辑
       public void setName(String name) {
           if (name != null && !name.trim().isEmpty()) {
               this.name = name;
           } else {
               System.out.println("姓名不能为空！");
           }
       }
       public void setAge(int age) {
           if (age > 0 && age < 150) {
               this.age = age;
           } else {
               System.out.println("年龄输入不合法！");
               // 可以选择抛出异常或赋予默认值
           }
       }
   }
   ```
4. **分析代码**：
   - 外部程序不能再用`对象.name = "张三"` 或 `对象.age = -10`直接访问和修改，必须通过`setName("张三")`和`setAge(20)`。
   - 在`setAge`方法中，可以校验数据的合法性，例如只允许设置0-150之间的值。
   - 封装的核心在于：**把数据和对数据的操作捆绑在一起，并对数据的访问进行控制**。

*时长：17 分钟*

## 巩固练习

（实践任务：设计与实现一个简单的封装模型）
1. **任务发布（1分钟）**：请设计并实现一个`Book`类。一本书有书名（title）、作者（author）、价格（price）三个核心属性。要求使用封装特性。
2. **学生实践（8分钟）**：学生根据刚才的讲解，在IDE中创建`Book`类。要求：
   - 将三个属性（title、author、price）声明为`private`。
   - 为每个属性编写对应的public的getter和setter方法（共6个方法）。
   - **挑战**：在`setPrice`方法中，要求价格（price）必须是一个正数（大于0）。如果设置的价格不合法，则方法什么也不做（或者打印一句提示）。
3. **巡堂指导（期间进行）**：教师巡堂，观察学生代码实现情况，对常见的错误（如方法命名不规范、访问修饰符使用错误等）进行即时个别指导。
4. **简单汇报与讲解（4分钟）**：请1-2位学生分享自己的代码（可通过投屏），教师点评并强调要点，展示一版参考实现代码。

*时长：13 分钟*

## 小结作业

1. **课堂小结（3分钟）**：教师带领学生回顾本节课要点：
   - 封装的目的是什么？（隐藏实现细节，保护数据，提供统一接口）
   - 在Java中如何实现封装？（两步走：用`private`隐藏属性；用`public`的getter/setter方法作为公共接口）
   - 封装带来的好处？（提高代码的安全性、可维护性和可复用性）
2. **布置作业（2分钟）**：
   - **必做题**：完成课上的`Book`类，并编写一个测试类`TestBook`，在其中创建Book对象，通过setter方法设置属性值，再通过getter方法获取并打印出来。要求对合法值和非法值（如负价格）都进行测试。
   - **思考题（选做）**：除了使用`private`和getter/setter，Java还有哪些方法/语法特性可以用来实现封装（提示：可以查找一下`包（package）`和访问控制修饰符的其他成员，如`protected`）？封装与“安全性”的关系是什么？
   - **预习任务**：预习面向对象的另一个核心特性——“继承”，思考封装好的类如何能被复用和扩展。

*时长：5 分钟*
